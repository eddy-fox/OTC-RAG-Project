# OTC Drug Document-based RAG System

## Overview
OTC 의약품 첨부문서를 검색 근거로 활용하는 RAG 기반 QA 백엔드 프로젝트입니다.
질문에서 약물명을 식별하고 관련 문서를 검색한 뒤, 검색 결과를 OpenAI API에 전달해 근거 기반 응답을 생성하도록 구성했습니다.
RAG 로직은 별도 모듈로 분리해 프론트엔드와 연동 가능한 구조로 설계했습니다
<br><br>
## Example
**Question**
`3세 아동한테 타이레놀 먹여도 돼?`

**Candidates**
- 우먼스타이레놀정
- 타이레놀콜드-에스정
- 어린이타이레놀현탁액(아세트아미노펜)
- 타이레놀8시간이알서방정(아세트아미노펜)
- 타이레놀산500밀리그램(아세트아미노펜)
- 타이레놀정500밀리그람(아세트아미노펜)
- 어린이타이레놀산160밀리그램(아세트아미노펜)

**Selected**
- 타이레놀정500밀리그람(아세트아미노펜)

**Answer**
결론: 주의 필요

이유: 타이레놀은 만 12세 이상 소아 및 성인에 적합하다고 명시되어 있으며, 3세 아동에게는 권장되지 않습니다. 이 약은 가능한 최단기간 동안 최소 유효용량으로 복용해야 하며, 아동에게 복용 시 의사의 조언을 반드시 받아야 합니다.       

근거: [문서1] "1회 1~2정씩 1일 3-4회 (4-6시간 마다) 필요시 복용한다."
      [문서1] "이 약은 가능한 최단기간동안 최소 유효용량으로 복용한다."

## Notes
- 문서 청킹시 용법용량의 길이는 적당해서 문제가 없었지만, 주의사항의 길이가 긴 문서가 많았음
  - 주의사항을 대분류 3개씩 묶어 청킹 → 여전히 길이가 길어 llm모델의 최대 컨텍스트 길이 제한에 걸림
  - 대분류를 한 단위씩 다시 청킹
- 사용자가 제품명을 정확히 입력하지 않거나 비슷한 약물을 혼용하는 경우가 많음.
  - 그대로 검색에 사용하면 정확도가 떨어질 것이기 때문에 '질문에서 약물명 후보 추출 → 사용자가 후보 선택 → 해당 약물명으로 검색 수행' 의 구조를 사용
- 응답 생성은 검색된 문서를 그대로 전달하지 않고 필요한 문맥 정리 후 OpenAI API에 전달하는 구조로 구성함.
- 주의사항과 용법용량만 사용했고 효과/효능에 대해서는 학습하지 않았기 때문에 특정 증상에 특정 약물을 복용해도 되는지와 같은 내용 검색이 불가. 추후 보완하여 업데이트 예정
- 일부 질문들에서 더 정확한 근거로 쓰일 수 있는 문서가 존재함에도 경고, 금기, 심각한 부작용을 명시하는 문서를 우선적으로 선택해서 답변의 중심 근거로 사용하는 경향 확인.
  - 이 문제는 프롬프트를 여러번 변형하여 테스트한 이후에도 반복됨.
  - 정확한 답변을 위해서는 단순 대분류단위 청킹을 보완하여 문장 임베딩 유사도에 기반한 의미 단위 분할이 필요함
 - 2가지 이상 약물을 함께 복용하는 것에 관한 내용은 학습하지 않아 대답이 불가함.
   - 추후 메타데이터에 존재하는 주성분명과 약물명을 매핑하여 각 약물의 성분과 계열 정보를 주입한 후
   - 주의사항 문서 내 병용금지에 관한 부분에서 검색한 약물의 주성분과 연결하여 같은 의미로 맞춰주는 단계를 추가하면 해결이 가능할 것이라고 추측. 추후 업데이트에 반영 예정
- 프로젝트를 진행하며 RAG의 품질은 문서 정제와 청킹에 생각보다 크게 영향을 받기 때문에 설계 초반이 가장 중요하다는 점을 확인했음
